## 报告漏洞

我们非常重视项目的安全性。如果你发现了安全漏洞，请按照以下流程报告：

### 报告方式

**请不要通过公开的 GitHub Issues 报告安全漏洞。**

请通过以下方式私下报告：

1. **GitHub Security Advisories（推荐）**
   - 访问 [Security Advisories](https://github.com/bsi-bcp/bcp-admin-ui/security/advisories)
   - 点击 "Report a vulnerability"
   - 填写漏洞详情

2. **电子邮件**
   - 发送邮件至项目维护者
   - 邮件标题请注明 `[SECURITY]`

### 报告内容

请在报告中包含以下信息：

- **漏洞类型**：例如 XSS、SQL 注入、CSRF 等
- **漏洞位置**：受影响的文件路径和代码行数
- **复现步骤**：详细的复现步骤
- **影响评估**：潜在的安全影响
- **概念验证**：如果可能，提供 PoC 代码或截图
- **修复建议**：如果有，提供修复建议

### 响应时间

| 阶段 | 时间 |
|------|------|
| 确认收到 | 48 小时内 |
| 初步评估 | 7 天内 |
| 修复发布 | 根据严重程度，通常 30 天内 |

### 处理流程

1. **确认接收**：我们会在 48 小时内确认收到你的报告
2. **评估验证**：我们会评估漏洞的有效性和严重程度
3. **制定修复**：开发并测试安全补丁
4. **发布更新**：发布包含修复的新版本
5. **公开披露**：在修复发布后，协调公开披露

### 严重程度分级

| 级别 | 描述 | 响应时间 |
|------|------|----------|
| 严重 | 可远程利用，无需认证 | 24-48 小时 |
| 高 | 可导致敏感数据泄露 | 7 天 |
| 中 | 需要特定条件才能利用 | 14 天 |
| 低 | 影响有限 | 30 天 |

## 安全最佳实践

### 部署建议

1. **环境变量**
   - 不要在代码中硬编码敏感信息
   - 使用环境变量管理配置

2. **HTTPS**
   - 生产环境必须使用 HTTPS
   - 配置正确的 SSL/TLS 证书

3. **API 代理**
   - 使用 Nginx 反向代理后端 API
   - 不要直接暴露后端服务

4. **依赖更新**
   - 定期更新依赖包
   - 使用 `npm audit` 检查已知漏洞

5. **访问控制**
   - 正确配置角色权限
   - 遵循最小权限原则

### 开发建议

1. **输入验证**
   - 对所有用户输入进行验证和过滤
   - 使用白名单而非黑名单

2. **XSS 防护**
   - 使用 Vue 的模板语法自动转义
   - 避免使用 `v-html` 处理用户输入

3. **CSRF 防护**
   - 确保后端正确配置 CSRF Token

4. **敏感数据**
   - 不要在前端存储敏感信息
   - 使用 HttpOnly Cookie
